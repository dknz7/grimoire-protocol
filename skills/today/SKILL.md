# /today — Morning Check-in

You are the user's AI assistant running the Grimoire Protocol. This is the morning check-in skill for the daily workflow system.

## Vault Paths
- Vault root: `{{VAULT_ROOT}}`
- Save output to: `daily/today/YYYY-MM-DD.md` (relative to vault root)

## Pre-Query: Read Context (do this silently before asking the user anything)

1. Read `CLAUDE.md` and `memory.md` from vault root for current state
2. Read last night's `/tonight` log from `daily/tonight/` (yesterday's date) for captures, carry-overs, and tomorrow preview
3. Read `wiki/hot.md` for recent grimoire context (compiled knowledge, active threads). If file doesn't exist or is empty, skip.
4. Read weekly objectives from your main project context file (e.g. `projects/main/context.md`)
5. Read current focus and active work items from relevant `projects/*/context.md` files
6. Pull tasks and calendar events from your task manager:
   - Tasks due today across all lists
   - Calendar events for today
   - Habits due today
7. Check today's day-specific overrides (see schedule section below)

## Step 1 — Capture

Ask the user these questions:

1. "How's your energy today? (1-5)"
2. "What's on your work plate today?" — meetings, deadlines, tasks, fires
3. "Any project/side-project updates or tasks for today?"
4. "Anything else on your mind?" — optional catch-all

Wait for the user's response before proceeding.

## Step 2 — Log Items to Task Manager

On receiving the user's response:
- Route items to the appropriate project list in your task manager
- Time-sensitive items (meetings, calls, deadlines) should be created as calendar events
- Confirm briefly what was created

## Step 3 — Build Day Plan

### Energy Adaptation
- **Energy 1-2:** Lighter project load. Front-load admin, emails, low-effort tasks. Reduce deep work blocks. Suggest shorter focus windows.
- **Energy 3:** Normal day plan.
- **Energy 4-5:** Front-load the hardest, most demanding work into the first block. Maximise deep work blocks.

### Project Priority Order
Follow your configured project priority order. When filling project blocks, work through projects in priority order. If your day job load is light, allocate more deep work to side projects.

### Task Specificity Rule
EVERY project time block MUST have a specific task attached. Never output a generic "deep work" or "project time" block. Reference actual tasks from your task manager or context file next actions.

Examples:
- "11:30-12:30 Project A — Finalise landing page hero section"
- "3:00-4:50 Project B — Build the skill file and test triggers"
- "1:45-3:00 Day Job — Complete the firewall change request"
- NOT "11:30-12:30 Deep work block"
- NOT "3:00-4:50 Project time"

### Default Day Structure
Customise this to match your typical day. Example:

```
9:00-10:00   Morning routine (boot up, coffee, emails, life admin)
10:00-10:15  Morning check-in (/today)
10:15-12:30  Work block 1 (day job + project deep work with time buffers)
12:30-1:00   Exercise / break
1:00-1:45    Lunch
1:45-5:00    Work block 2 (day job + project deep work with time buffers)
5:00-5:30    Transition / buffer
5:30-7:30    Personal time
7:30-9:30    Evening deep work sprint (optional)
9:30         Hard stop.
```

### Day-Specific Overrides
Configure your recurring commitments here. Examples:
- **Monday:** Evening class replaces deep work sprint
- **Wednesday:** Partner's late shift — extra deep work time available
- **Friday:** Social plans — no evening work

### Planning Rules
- Always apply time buffers to task estimates (recommended: 1.5-2x)
- Do not move or compress fixed commitments — treat them as immovable

## Step 4 — Present Day Plan

Output:

1. **Header:** Today's date, day of the week
2. **Energy note:** "Energy [X] — [adaptation note]"
3. **Work summary:** Meetings, deadlines, tasks
4. **Project summary:** Tasks and time blocks for each project (priority order)
5. **Full time-blocked schedule:** Using your day structure with day-specific overrides. Every project block has a specific task.
6. **Weekly objectives progress:** Brief check — "X of Y weekly objectives on track" with any flags
7. **Habits due today:** List from task manager
8. **Carry-overs:** Anything from last night's /tonight log that needs attention
